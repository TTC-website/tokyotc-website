## 2. 決まっている方針（合意済み）

1. **置き場所＝第一階層 `/sake/`。** 体験メニュー（`/experience/`）の配下ではない。既存サイトが `/insurance/ /inheritance/ /property/ /konkatsu/ …` と第一階層にセクションを並べる構造なので、それに揃える。SEO（ドメイン権威の共有）の観点でもサブディレクトリが最有利。
2. **デザイン方針 =(い)「見た目を寄せて、きれいに作り直す」。** 元サイトのスクリーンショットを“正”として見た目を再現しつつ、コードはクリーンに新規で書く。Google Sites の難読化HTMLはそのまま使わない（使えない）。
3. **`/experience/sake-event/` は触らない。** これは体験メニューの個別ページ（7体験のうちの1つ）で、`/sake/`（日本酒の会の独立サイト）とは**別物**。混同・リダイレクト・上書きをしないこと。
4. トップページのデザインは**小西さん承認済み（2026-06-09）**。この見た目に全ページを統一する。

## 7. 技術メモ

- 構成：静的HTML（`public/<section>/index.html`）。Vite + Netlify。
- ビルド：`npm run build` → `dist/`。ローカル確認：`npm run preview` → `http://localhost:4173/`
- `/sake/index.html` は**素のHTML/CSS（Reactなし）**。クリーンで運用しやすい。新規ページもこの方式で。
- 画像の相対参照に注意：
  - トップ（`/sake/index.html`）→ `assets/xxx.jpg`
  - 個別ページ（`/sake/reports/2026-01-09/index.html`）→ `../../assets/xxx.jpg`
- `public/_redirects`（Netlify）は konkatsu のみ。`/sake/` は新規セクションなのでリダイレクト不要。
- `public/sake/assets/` の画像一覧：logo-2026 / sake-bg-records / sake-bg-strip / sake-bottles-blue / sake-garden-day / sake-hero-bottles / sake-venue-entrance / sake-venue-night / sake-venue-room / sake-venue-sign / sake-warmer

## 8. デザインの基準（`/sake/index.html` に準拠）

- CSS変数：`--ink:#303030` / `--bg:#fff` / `--soft:#f6f5f2`（セクション交互背景）/ `--line:#e5e7eb` / `--accent:#3a4a3a` / `--sans`（ゴシック系）
- ヘッダー：`position:fixed`・ダーク半透明（`rgba(24,26,24,.42)` + blur）・白文字
- ヒーロー：フルワイド写真＋中央の大きな白文字タイトル（暗いグラデを重ねて可読性確保）
- セクション：白 ↔ `--soft` を交互。`.wrap`（最大1100px）、本文は `.lead-narrow`（720px）
- ギャラリー：`.gallery` 非対称グリッド（モバイルで2列）
- ボタン：`.btn`（枠線→ホバーで塗り）
- **レポート・図鑑も、必ずこの見た目に統一する**（第3回の旧ダーク版は作り直す）
